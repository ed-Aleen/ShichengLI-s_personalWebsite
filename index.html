    Project Description

    The purpose and objective of our Food Logger mobile application is to address the growing challenge of maintaining healthy dietary habits amidst the demands of urban life. By offering users a convenient platform to set and track their dietary goals, this application empowers individuals to make informed choices about their food intake. Through features such as personalized profiles, goal setting functionalities, and a comprehensive shared food database in order to incentivise consumer collaboration and community, users can effortlessly monitor their nutritional intake throughout the day, week, month, and beyond. Additionally, the incorporation of gamification elements adds a motivational aspect, encouraging users to stay committed to their goals by incentivising them with ‚Äúgummies‚Äù which is a currency they can use to open collectable card packs, which they can then collect. Ultimately, the application serves as a valuable tool for promoting healthier lifestyles and fostering a sense of accountability and achievement among its users, all from the tip of their handheld phones. Our Food Logger mobile application is built upon a full-stack system. The frontend is developed using React Native, leveraging TypeScript for enhanced type safety and developer productivity. React Native enables us to create a cross-platform application, ensuring compatibility with both iOS and Android devices while maintaining a single codebase. On the backend we utilized node.js in order to handle server-side operations. MongoDB serves as our database solution, providing flexibility and scalability for managing user, food, meal, goal, and gamification related documents.

    Notable Features

    • Integrated personalized profiles and a comprehensive shared food database into the application
    • Implemented user registration and login, creation, setting, and tracking of personal profiles, and dietary goal management
    • Users can record each meal from the shared database, expand the food database, and customize food attributes, with food record analysis views provided
    • Utilized gamification elements to motivate users to achieve their goals, maintain healthy eating habits, and enhance their sense of responsibility and achievement

    During the project's testing phase, Jest framework was used to achieve a test coverage of 95% for both frontend and backend code.

    Pull Requester

    ×

    Inspiration

    When I code, I'm not a fan of writing pull requests. Think about the constantly looking through each line you changed and trying to figure out how should you make it clear for other teammates to know what you did (and like what you did).

    What it does

    Writes pull requests comments for you.

    How we built it

    We used OAuth to authenticate users through GitHub and give permissions for repository access. After that, we read off the URL and use the GitHub API to request for the difference of the two branches. We use that difference information, reformat it, and feed it to LLM on VectorShift to generate pull request bodies.

    Challenges we ran into

    The two biggest challenges were:

    1. We couldn't figure out how OAuth works, and how to accommodate it into Chrome Extension.
    2. We had little knowledge about JavaScript, which was what we used to build the extension, making it fairly challenging to debug.

    Accomplishments that we're proud of

    We overcame the two biggest challenges, we got OAuth sorted out, and we also finished the project with technologies we weren't familiar with before.

    What we learned

    We learned a lot of technical knowledge which without saying. We also learned how to work together as a small team, and in such a short duration.

    What's next for LewLiOs KeHui

    The extension has been submitted for publish reviewing, coming soon!

    Autonomous Home Monitoring Robot

    ×

    Project Description

    This project revolves around a TurtleBot3 robot designed to uphold specific household regulations during gatherings, ensuring a safe and organized atmosphere. It takes charge of patrolling rooms A, B, and D, with a focus on enforcing two primary rules: restricting human access to the kitchen (Room D) and preventing any mingling between the family cat and dog.

    Notable Features

    • ROS Service Integration: Allows the robot to navigate through designated areas, offering flexibility in patrol routes.
    • Actionlib Server: Streamlines decision-making by executing action-based directives, particularly in enforcing established rules.
    • SMACH State Machine: Oversees various tasks undertaken by the robot, including navigation, rule enforcement, and utilizing Text-to-Speech technology for communication.
    • YOLO Detection: Utilizes the YOLO algorithm for instant identification of rule violations during navigation, prompting immediate corrective actions.
    • Feedback and Logging: Provides ongoing feedback on rule breaches and maintains a record of infractions for thorough evaluation.

    This system epitomizes an intricate fusion of robotic autonomy, real-time detection algorithms, and prompt enforcement protocols, presenting an innovative solution for upholding decorum during social gatherings. Witness the project in action here.
